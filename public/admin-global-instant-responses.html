üí° Testing Tips & Best Practices

üéØ Trigger Matching:

  • Triggers are matched using keyword matching (not exact phrases)
  • Order doesn't matter: "hold please" = "please hold"
  • Case-insensitive: "HOLD" = "hold" = "Hold"

üö´ Negative Triggers:

  • If ANY negative trigger appears, scenario will NOT match
  • Use to prevent false positives: "don't hold" should NOT trigger "Hold Request"

üîß Regex Triggers:

  • Advanced pattern matching for power users
  • Example: \b(hold|wait)\s*(on|up)?\b matches "hold", "wait", "hold on", "wait up"
  • Use for phone numbers, dates, specific patterns

üìä Confidence & Priority:

  • Min Confidence: How sure the AI must be (0.7 = 70% confident)
  • Priority: Tie-breaker if multiple scenarios match at same confidence
  • Emergency scenarios: high confidence (0.9+) and high priority (8-10)
  • Chitchat scenarios: low confidence (0.5) and low priority (-3 to 0)
