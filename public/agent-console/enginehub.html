Engine Control

Master switch and operating mode for the conversational intelligence engine.

When off, the engine does not influence any call routing. All calls use existing flow logic unchanged.

Controls how the engine interacts with live calls.

Intent Detection

When on, the engine detects all intents in one utterance simultaneously. When off, only the primary intent is acted upon.

Policy Router

Select which policies this company's engine is allowed to use. Policies not enabled here are never selected regardless of intent match.

answer_then_book Answer the KC knowledge question first, then immediately move to booking the appointment.
book_then_defer Confirm the booking first, then note the secondary question and address it after confirmation is complete.
clarify_first Ask one focused clarifying question before acting. Used when intent confidence scores are too close to distinguish.
pause_resume Handle a mid-flow injection — answer the question, then resume the active flow from its last checkpoint without re-asking captured data.
de_escalate When a caller is distressed, validate their concern and redirect to a concrete action before climbing the escalation ladder.
offer_alternatives When a human is unavailable, offer the caller meaningful alternatives: voicemail, callback scheduling, or service appointment.
transfer_gate Before connecting to a live human, confirm with the caller that this is what they want. Reduces unnecessary transfers.
transfer Connect the caller to an available live human agent. Only fires after gate check confirms availability and caller intent.
take_message Capture caller contact information for a callback. Used after-hours or when no one is available to take the call.

Knowledge Engine

Controls how strictly the engine grounds its responses in Knowledge Cards. Grounded mode prevents the agent from inventing pricing, policy, or scheduling commitments.

When on, the agent may only answer from KC cards. It never generates pricing, policy, service area, or scheduling commitments from memory. Strongly recommended.

What the engine does when the caller asks something and no Knowledge Card matches the intent.

Log all no-match events to the call trace. These appear in Call Intelligence as a KC Gap Report — shows exactly which questions the agent couldn't answer from your Knowledge Cards.

Agenda State

Replaces the single SPFUQ anchor. The engine now tracks all open caller intents across the full call — pending, deferred, and resolved. No question gets dropped silently.

Maximum number of intents tracked as deferred at any one time. When the limit is hit, the oldest deferred intent is dropped.

When on, the agent automatically raises deferred intents at the next natural opening in the conversation. The caller never has to repeat themselves.

Drop a deferred intent from the agenda after N turns without surfacing it. Prevents the agent from bringing up stale topics at the wrong moment.

Escalation Ladder

Four rungs. The agent climbs one rung at a time — it never jumps straight to a live transfer. Each rung is independently configurable.

1
De-Escalate + Redirect Caller is distressed but has not asked for a human. Agent validates the frustration and redirects to a concrete action.
2
Gate Check Caller has explicitly asked for a human. Engine checks: are we open? Is someone available? What does this company prefer?
Always runs
3
Offer Alternatives Human is not available. Offer the caller meaningful options. Select which alternatives this company supports.
4
Confirm Then Transfer Human is available. Before connecting, the agent confirms with the caller that this is what they want.
