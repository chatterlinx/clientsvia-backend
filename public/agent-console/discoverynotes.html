Two-Phase Entity Model v2 Architecture

Everything the agent hears goes into temp{} immediately — unverified, living hypothesis. At UAPB BOOKING_CONFIRM, when the caller says "yes" to the booking readback, all temp fields lock simultaneously into confirmed{}. The confirmed{} snapshot is what powers caller charts, handoff context, and call history.

🔵 TEMP{} — Working Hypothesis
firstName lastName fullName address phone email issue serviceType staffMentioned preferredDate preferredTime confidence{}

Agent accumulates freely every turn.
Never locked mid-call. Never re-asked once present.

⚡
UAPB BOOKING CONFIRM
One "yes" locks everything
🟢 CONFIRMED{} — Locked Truth
firstName lastName fullName address phone issue serviceType preferredDate preferredTime

Locked at booking confirmation only.
Powers caller charts + handoff context.
