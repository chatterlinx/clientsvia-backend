🧠 discoveryNotes — Full Schema & Status

Redis key: discovery-notes:{companyId}:{callSid} · TTL=4h
MongoDB: Customer.discoveryNotes[] · fire-and-forget mid-call, full $push at call end
Wire point (read): HybridReceptionistLLM.js · KCDiscoveryRunner.js (Gate 2)
How it works: On Turn 1, the LLM intake extracts entities from the caller's first utterance and writes them to Redis. Every subsequent turn reads this live state before Groq is called, so the agent never re-asks what it already knows and can answer in context (e.g. "Caller is calling about annual maintenance — Caller's name is Mark").
