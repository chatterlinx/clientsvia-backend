ⓘ How the Arbitration Pipeline Works

Platform Tier (locked) — Built-in interceptors that fire first on every call. The Greeting Interceptor handles turn 1. Promotions and Pricing are anti-hallucination guardrails that read from MongoDB. Their signals can be extended per company.

Rule Intents (your custom rules) — Keyword-based rules you build here. Evaluated in priority order after platform rules. First match wins. Use rules to give instant, guaranteed answers for specific phrases — no AI scoring needed.

KC Semantic Engine — Scores all Knowledge Containers against the utterance. Fires when no rule matches. Maintains topic anchoring across turns. Fast lane via Groq.

LLM Fallback → Graceful ACK — Claude handles anything KC cannot confidently answer. Graceful ACK is the terminal catch-all.

ⓘ Arbitration Policy — How Decisions Are Made

When a caller utterance contains multiple signals (e.g. booking intent + pricing keyword), the Arbitration Engine scores all candidates simultaneously and applies this policy to decide what wins. No more first-match-wins. No more pricing hijacking booking turns.

Signal Weights control how much each signal type contributes. Pricing weight (0.50) is intentionally lower than Booking (0.90) — this is why "I'd like to schedule, how much is it?" correctly routes to booking, not pricing.

Lane Locking keeps the agent inside the active task (booking, discovery) across multiple turns — exactly how Rasa CALM sticky routing works. Escape keywords let callers override at any time.
